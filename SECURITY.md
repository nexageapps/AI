## Reporting a Vulnerability

If you believe you have found a security vulnerability in this project, please
report it using one of the following private channels:

- Email: `security@example.com` with a clear subject line such as
  `"Security issue in <project-name>"`.
- If this repository is hosted on GitHub, use the **Security → Report a vulnerability**
  feature to open a private security advisory instead of creating a public issue.

When reporting, please include:

- A description of the issue and its potential impact.
- Steps to reproduce, proof-of-concept code, or sample input where possible.
- Any known workarounds.

We aim to:

- Acknowledge your report within **3 business days**.
- Provide an initial assessment or request for more information within **7 business days**.
- Keep you updated on the progress at least every **14 days** until the issue is resolved.

If the vulnerability is confirmed, we will work on a fix and coordinate a release
and disclosure timeline with you. If we determine that the reported issue does
not constitute a security vulnerability, we will explain why.
